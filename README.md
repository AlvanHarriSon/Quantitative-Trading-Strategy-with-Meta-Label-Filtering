## Overview

This project explores whether a **machine-learning-based meta-label filter** can improve a traditional rule-based trading strategy.

Instead of taking every signal generated by a primary strategy, the pipeline works in two layers:

1. **Primary Strategy**  
   Generate candidate long-entry events under trend, breakout, and volume-confirmed conditions.

2. **Meta-Label Model**  
   Learn from historical candidate events and estimate the probability that a signal should be taken.

3. **Backtest Comparison**  
   Compare:
   - Buy-and-hold benchmark
   - Primary strategy baseline
   - Meta-filtered strategy

The goal is not just signal generation, but building a more complete research pipeline with:

- feature engineering
- event-driven labeling
- walk-forward training
- realistic transaction cost assumptions
- risk control and cooldown logic
- strategy-vs-benchmark comparison

## Strategy Logic

### Primary Entry Conditions

A candidate long signal is generated only when all of the following conditions are met:

- bullish regime (`EMA50 > EMA200` and price above `EMA200`)
- breakout above rolling high
- trend strength confirmed by ADX
- volume confirmation via volume ratio threshold

To avoid repeated signals during consecutive days, only **setup transitions from `0 -> 1`** are treated as valid events.

### Exit Logic

Trades are managed using the same exit logic both for labeling and for backtesting:

- **ATR trailing stop**
- **Trend exit** after minimum holding bars
- **Time exit** after maximum holding bars
- **Max drawdown emergency control**
- **Cooldown period** after risk trigger

### Meta-Label Layer

After candidate events are generated:

- each event is labeled using realized trade outcome
- event features are extracted
- a **walk-forward logistic regression model** predicts whether a signal should be taken
- only signals above the configured probability threshold are executed in the meta-filtered strategy

This design helps separate:

- **signal generation**
- **signal selection**

which is a useful structure for systematic strategy research.

## Methodology

### Feature Set

The current model uses a compact feature space including:

```text
ret_10
rsi_14
macd_hist
atr_pct
adx_14
dist_ema20
ema50_ema200_spread
```

These features aim to capture:

- momentum
- volatility
- trend strength
- distance from short-term trend
- long-term regime structure

### Training Procedure

The project uses walk-forward retraining on the event dataset:

- split candidate events into train/test by time order
- retrain the classifier periodically
- predict only on future unseen events

This setup is closer to realistic research practice than random shuffling.
